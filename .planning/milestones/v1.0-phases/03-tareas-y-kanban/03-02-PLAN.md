       ```php
       public function index(Request $request)
       {
           $query = Task::with('client')
               ->when($request->filled('cliente'), fn ($q) => $q->where('client_id', $request->cliente))
               ->when($request->filled('estado'), fn ($q) => $q->where('estado', $request->estado))
               ->when($request->filled('prioridad'), fn ($q) => $q->where('prioridad', $request->prioridad))
               ->when($request->filled('titulo'), fn ($q) => $q->where('titulo', 'like', "%{$request->titulo}%"))
               ->latest()
               ->get();

           $columns = [
               'backlog'      => $query->where('estado', 'backlog')->values(),
               'en_progreso'  => $query->where('estado', 'en_progreso')->values(),
               'en_revision'  => $query->where('estado', 'en_revision')->values(),
               'finalizado'   => $query->where('estado', 'finalizado')->values(),
           ];

           return Inertia::render('Admin/Tasks/Index', [
               'columns'  => $columns,
               'clients'  => Client::orderBy('nombre')->get(['id', 'nombre']),
               'filtros'  => $request->only(['cliente', 'estado', 'prioridad', 'titulo']),
           ]);
       }
       ```
       IMPORTANT: The `$query->where('estado', ...)` calls after `->get()` are Collection methods, not Query Builder. Since `estado` is cast to TaskStatus enum, compare using the enum value string. If Collection `where` doesn't match due to cast, use `->filter(fn ($t) => $t->getRawOriginal('estado') === 'backlog')` or compare with the enum case. Test this during implementation and adjust. The safest approach: use `$query->groupBy('estado')` and then map keys, or group after converting to array. Alternatively, `$query->where('estado', TaskStatus::Backlog)` should work since the collection items have cast values.

       ```php
           return back();
       ```

       ```php
       {

           return back();
       }
       ```

       **destroy():** Delete the task, redirect back.
       ```php
       public function destroy(Task $task)
       {

           return back();
       }
       ```

       **updateStatus():** Separate endpoint for drag-drop. Validate `estado` inline, update, redirect back.
       ```php
       public function updateStatus(Request $request, Task $task)
       {
           $request->validate([
               'estado' => ['required', 'in:backlog,en_progreso,en_revision,finalizado'],
           ]);
           $task->update(['estado' => $request->estado]);
           return back();
       }
       ```

       - Simplest: Use `->toDateString()` in a resource or append it. For this project, handle in Vue: `task.fecha_limite?.substring(0, 10)`.
