## Spacing Scale

Declared values (multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline badge padding (`px-2 py-0.5`) |
| sm | 8px | Compact element spacing, gap between inline items |
| md | 16px | Default element spacing, section padding (`p-4`), row padding |
| lg | 24px | Section padding (`py-6`), between list sections |
| xl | 32px | Layout gaps, between major dashboard sections |
| 2xl | 48px | Major section breaks |
| 3xl | 64px | Page-level top/bottom spacing |

Exceptions:
- Sidebar width: 220px (fixed, per CONTEXT.md decision — not a spacing token)
- Sidebar item touch target minimum: 44px height (accessibility — icon + text row)
- Main content left offset: matches sidebar width exactly (220px)

**Source:** Existing codebase uses `py-6 px-4 sm:px-6 lg:px-8` on `<main>`, `mb-6` for section breaks, `p-3` on Kanban cards, `gap-4` on Kanban columns — all consistent with 4-point base scale.

## Component Inventory

### Compact Task Row

Each task in both sections renders as a single horizontal row inside a `bg-white rounded-lg` list container:

```
<div class="flex items-center gap-4 px-4 py-3 hover:bg-gray-50 cursor-pointer border-b border-gray-100 last:border-0">
  <span class="flex-1 text-sm font-medium text-gray-900 truncate">{{ task.titulo }}</span>
  <span class="text-xs text-gray-500 w-32 truncate">{{ task.client.nombre }}</span>
  <span class="px-2 py-0.5 text-xs font-medium rounded-full [priority-classes]">{{ task.prioridad }}</span>
  <span class="text-xs w-24 text-right [urgency-classes]">{{ task.fecha_limite }}</span>
  <select class="text-xs border-gray-300 rounded text-gray-700 py-0.5"><!-- status options --></select>
</div>
```

Column widths:
- Title: `flex-1` (fills remaining space), `truncate` for overflow
- Client name: fixed `w-32`, `truncate`
- Priority badge: intrinsic width (badge pattern from Phase 3)
- Date: fixed `w-24`, right-aligned
- Status dropdown: intrinsic width, minimal styling (`text-xs`)

### Status Dropdown (inline)

Use a native `<select>` element — no external library. Trigger is the select itself (no custom trigger). Options are all 4 enum values:

```html
<select @change="updateStatus(task.id, $event.target.value)" class="text-xs border-gray-300 rounded text-gray-700 py-0.5 bg-white">
  <option value="backlog">Backlog</option>
  <option value="en_progreso">En progreso</option>
  <option value="en_revision">En revision</option>
  <option value="finalizado">Finalizado</option>
</select>
```

On change: `router.put(route('tasks.updateStatus', task.id), { estado: newValue }, { preserveState: true, preserveScroll: true })`. Task disappears from list reactively when it transitions to `finalizado` or no longer meets section criteria.

## Interaction Contract

### Task row click
- Target: anywhere on the row except the status dropdown
- Action: `router.get('/tasks', { cliente: task.client_id }, { preserveState: true })`
- Navigates to Kanban filtered by client

### Status dropdown change
- Target: the `<select>` element in the row
- Action: `router.put(route('tasks.updateStatus', task.id), { estado: value }, { preserveState: true, preserveScroll: true })`
- On success: Inertia re-renders props — task disappears from section if it no longer qualifies (e.g. changes to `finalizado`)
- Stop propagation: `@change.stop` to prevent row click from firing

### Active nav state
- Sidebar items use `$page.url.startsWith(path)` to detect active route — same pattern as current `AdminLayout.vue`
